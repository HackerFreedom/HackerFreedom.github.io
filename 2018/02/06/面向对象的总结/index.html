面向对象的思想 （面试重点）

  1. 更符合我们的思维习惯
  2. 把复杂的问题简单化
  3. 把我们从执行者变为指挥者

面向对象的三大特性

1. 封装

1. 是用private修饰方法封装起来的类
    1. 作用：提高代码的复用性，和安全性

  • this关键字
    代表对象，就是所在函数所指的对象引用

    一般用于构造函数的引用
    调用格式 ： this. ()
    this对象后面跟上 .调用是成员属性或者是方法
    this（） 调用的是本类中对应参数的构造函数
    注意：用this调用构造函数，必须定义在构造函数的第一行。因为构造函数是用于初始化的，所以初始化动作一定要执行。否则编译失败。

  • static关键词

    • 作用：
      1. 想要实现对象中的共性数据的共享，可以将数据进行静态修饰
      2. 被static的成员方法，可以直接被类名直接调用，（类名. 静态方法）
      3. 静态随着类的加载而加载，且优先于对象的存在
    • 劣势：
      1. 静态方法只能访问静态成员，不可以访问非静态成员
      2. 静态方法中不能使用this，super关键字（this代表是对象，而在静态时，有可能就没有对象）
        成员变量和静态变量区别
    1. 位置不同
      1. 成员变量存在堆内存中，而静态变量存在于方法区中
    2. 生命周期不同
      1. 静态变量随着类的加载而存在，随着类的消失而消失
      2. 成员变量随着对象的创建而存在，随着对象被销毁而销毁

2. 继承

1. 子类可以访问父类的非私有成员变量和方法
- 作用：    
    1.提高代码的复用性，
    2. 使类和类之间产生了关系，是产生多态的一个前提
  • 继承特点
  1. 单继承： 一个类只能有一个父类
  2. 多重继承： 一个父类可以被多个子类继承 A继承B B继承C C继承D。

super关键词
用来调用父类中的属性值

  • 为什么在子类的构造函数的第一行总是会加入super（）？
    在子类继承父类，会继承到父类的数据，所以必须要父类先要有数据的初始化，
    所以子类在进行对象初始化的时候，要先调用父类的构造函数，这就是子类的实例化过程

  • 注意
    super在每一个子类的构造方法中是默认在第一行的。
    且和this关键字，两者只能出现一个。一山容不下二虎的!!

    3. 多态

    1. 函数就有多态性， 就是某一事物不同的具体表现
    • 作用
      1. 提高了程序的扩展性
    • 前提
      1. 必须要有实现关系或者继承关系
      2. 必须要有方法的重写

  • 方法的重写
    1、方法名、参数、返回值相同。

　　 2、子类方法不能缩小父类方法的访问权限。(public>protected>default>private）

　　 3、子类方法不能抛出比父类方法更多的异常(但子类方法可以不抛出异常)。

　　 4、存在于父类和子类之间。

　　 5、方法被定义为final不能被重写。

final 特点

  1. 这个关键词是一个修饰符，可以修饰类，方法，变量
  2. 被final修饰的类是一个最終类，不可以被继承
  3. 被final修饰的方法是一个最终的方法，不可以被重写

  4. 被final修饰的变量是一个常量，只能赋值一次
    命名的规范：常量名称定义时，有规范，所有字母都大写，如果由多个单词组成，中间用 _ 连接。

    • 方法的重载
      方法重载是让类以统一的方式处理不同类型数据的一种手段。多个同名函数同时存在，具有不同的参数个数/类型。
      重载Overloading是一个类中多态性的一种表现。
      1、参数类型、个数、顺序至少有一个不相同。

    　　2、不能重载只有返回值不同的方法名。

    　　3、存在于父类和子类、同类中。

    • 规则:
    1. 必须具有不同的参数列表；

  1. 可以有不同的返回类型，只要参数列表不同就可以了；

  2. 可以有不同的访问修饰符；

  3. 可以抛出不同的异常；

抽象类

  1. 定义：
    1. 被abstract所修饰的类，
  2. 特点：
    1. 抽象方法只能定义在抽象类中， 抽象类也可以有非抽象的方法
    2. 抽象类不可以被创建对象（实例化）
    3. 只能通过子类继承或者实现的方法，才能使子类实例化

接口

  1. 定义：
    比抽象类还抽象的类，用关键词interface修饰
  2. 接口的特点：
    1. 提高代码的可扩展性
    2. 规范并且约束了子类的行为（功能）
  3. 接口中的成员的特点：
    1. 成员变量：
      接口中没有成员变量,只能由自定义常量; –>因为接口中的方法没有方法体,所以没有地方给变量赋值;
      自定义常量的定义格式:
      public static final 数据类型 常量名 = 初始化值;
    2. 成员方法：
      默认被 public abstract 所修饰;
      接口中的成员方法全部都是抽象方法;
    3. 构造方法：
      接口中没有构造方法,所以接口不能创建对象;

  4. 接口和抽象类的区别:

    1.区别1:定义不同
    接口:比抽象类还抽象的类;
    抽象类:被abstract所修饰的类;
2.区别2:构造方法    
    接口:没有构造方法,不能创建对象
    抽象类:有构造方法,不能创建对象,但是可以给成员变量初始化;
3.区别3:成员变量    
    接口:接口中没有成员变量只有自定义常量;
    抽象类:可以有成员变量也可以有自定义常量;
4.区别4:成员方法    
    接口:接口中的成员方法全部都是抽象方法;默认被 public 和 abstract  所修饰;
    抽象类:可以有抽象方法也可以有非抽象方法;
5.区别5:和类之间的关系:
    接口:接口和类之间的关系是实现关系: implements
    抽象类:抽象类和类之间的关系是继承关系: extends

    总结各个类于接口的关系
    接口和类之间的关系:

    实现关系: implements
格式:  public class 类名 implements 接口名1,接口名2....{
            //实现接口中的抽象方法
        }
特点: 多实现,多层实现

    接口和接口之间的关系:

    继承关系: extends
格式:public interface 接口名 extends 接口名1,接口名2...{

        }
特点:多继承,多层继承

    类和类之间的关系:

    继承关系: extends
格式:
    public class 类名 extends 类名1{

    }
特点:单一继承,多层继承;

代码块

  1. 局部代码块

    1. 位置：
      1. 方法内，for循环的大括号内，方法声明上
    2. 作用：
      1. 用于限定变量和对象的生命周期和作用域

  2. 构造代码块

    1. 位置：
      1. 成员位置-类中方法外
      2. {
        //在类中方法外的代码块是构造代码块;
        }
    2. 作用：
      1. 抽取构造方法的共性，在每次创建对象前执行一次
      2. 执行次数:每创建一次对象就执行一次构造代码块;
        执行顺序:因为是对象产生的预加载;
        先执行构造代码块再执行构造方法;

  3. 静态代码块

    1. 位置：
      1. 成员位置，类中方法外 且被static所修饰的
    2. 作用：
      1. 类的预加载，随着类的加载而加载，且执行一次
        static{
        //在类中方法外并且被static修饰的代码块是静态代码块
        }
        执行顺序:
        静态代码块 –> 构造代码块 –> 构造方法

  4. 同步代码块（多线程）
    同步代码块的格式：
    synchronized(同步对象){
    需要同步的代码；
    }s

作用:
维持多个线程在执行任务时的秩序,当一个线程没有完成任务的时候,另一个线程不能进入同步代码块;
线程锁:
这个锁对象可以是任意类型的对象,但是这个锁对象必须被所有的线程对象所共享;–>必须在成员位置;
一般情况下我们会在锁对象的位置上写一个this;

同步方法:被synchronized所修饰的方法是同步方法;
同步方法也是有锁对象的;
非静态的同步方法的锁对象就是this;
静态的同步方法的锁对象是当前类的字节码对象;
