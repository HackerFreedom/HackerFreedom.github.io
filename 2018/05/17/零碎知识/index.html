redis 和 session 存入数据的区别

1.与其他用户状态保存方案比较
一般开发中用户状态使用session或者cookie，两种方式各种利弊。

Session:在InProc模式下容易丢失，并且引起并发问题。如果使用SQLServer或者SQLServer模式又消耗了性能

Cookie则容易将一些用户信息暴露，加解密同样也消耗了性能。

Redis采用这样的方案解决了几个问题，

  1. Redis存取速度快。

  2. 用户数据不容易丢失。

  3. 用户多的情况下容易支持集群。

  4. 能够查看在线用户。

  5. 能够实现用户一处登录。（通过代码实现，后续介绍）

  6. 支持持久化。（当然可能没什么用）
